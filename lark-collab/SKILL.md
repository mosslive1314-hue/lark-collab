# lark-collab v1.0.0 - 飞书协作流

> **飞书群聊协作全流程自动化基座** — 讨论交给人，整理交给机器。从讨论到落地，完整闭环自动化。

## 🎯 定位

**lark-collab 是一个**飞书协作自动化通用基座：

- 你在飞书群聊讨论完，它自动帮你**整理信息 → 生成文档 → 写入表格 → 创建会议 → 分配任务 → 发送提醒**
- 利用飞书原生能力（文档/多维表格/日历/视频会议/机器人），不重复造轮子
- **减少手动重复工作**，让你专注讨论，不用事后整理
- 通用基座设计，**支持任意协作场景**，内容完全由你讨论决定

**适合场景**：
- ✅ 团队选题讨论 → 自动整理选题库 → 分配任务 → 创建截稿日历
- ✅ 项目会议结束 → 自动整理会议纪要 → 提取待办 → 创建跟进日历
- ✅ 约团队会议 → 确定时间后**一键创建飞书视频会议** → 邀请所有人 → 发送会议链接
- ✅ 每日行业信息收集 → 自动分类整理 → 生成周报文档
- ✅ 社群活动收集 → 自动整理活动信息 → 写入多维表格 → 创建活动日历提醒
- ✅ **任何需要「讨论 → 整理 → 执行」的协作场景**都能用

## 🧠 策略引擎

### 核心工作流（完整闭环，选择权留给用户）

```
  群聊讨论完成（会议/选题/项目讨论）
    ↓
@lark-collab 触发整理
    ↓
  1. 读取讨论时间段的完整群聊历史
  2. AI 深度理解，提取结构化信息：
     - 讨论主题和结论
     - 需要跟进的待办任务
     - 需要召开的会议（时间/参会人）
     - 每个任务的责任人 + 截止日期
  3. 生成完整的飞书文档（会议纪要/选题列表）
  4. 结构化数据写入飞书多维表格（方便筛选跟踪）
  5. **列出提取结果，请求用户确认**：
     - "提取到以下会议/待办，是否创建日历？"
     - 如果责任人匹配不确定，先确认责任人
  6. **用户确认后**，再执行创建：
     - 会议 → 自动生成飞书视频会议链接
     - 截止日期 → 每个责任人不同时间分别创建提醒
  7. @ 相关责任人，发送提醒和链接
  8. 输出完整文档链接到群聊
    ↓
  ✅ 完成！从讨论到落地，全闭环自动化，但是决定权始终在你
```

## ✨ 核心能力

| 能力 | 说明 |
|------|------|
| 📖 **读取群聊历史** | 获取指定时间段/条数的群聊讨论上下文 |
| 📝 **会议纪要自动整理** | 会议结束后，自动整理讨论结论、待办任务、决策事项 |
| 🧹 **自动结构化整理** | 从散乱聊天提取结构化信息：结论/待办/会议/时间/责任人 |
| 📄 **生成飞书文档** | 整理结果生成格式化飞书文档，可分享 |
| 📊 **写入多维表格** | 结构化数据存入多维表格，方便后续筛选跟踪 |
| ✅ **提取结果预确认** | 列出提取到的会议/待办，**请求用户确认后再创建**，决定权留给用户 |
| 📅 **确认后创建日历 + 视频会议** | 用户确认后，自动创建飞书日历，**自动生成飞书视频会议链接** |
| 👥 **任务自动分配** | 识别责任人，每个任务不同截止日期分别提醒，昵称不确定时主动请求确认 |
| 🔔 **自动提醒** | 会议开始前提醒，日历提前提醒，每个任务截止日期分别提醒 |

## 🛠️ 工具调用（通过飞书 CLI）

```bash
# 1. 读取群聊历史
lark-cli im messages list --chat-id {chat_id} --limit {limit}

# 2. 创建飞书文档
lark-cli doc create --title {title} --content {markdown_content}

# 3. 写入多维表格
lark-cli bitable record add --app {app_token} --table {table_id} --fields {json}

# 4. 创建日历 + 自动生成视频会议
# 默认自动创建飞书视频会议，不需要额外配置
lark-cli calendar event create --summary {summary} --start {start} --end {end} --attendees {emails} --vchat

# 5. 发送群消息
lark-cli im message send --chat-id {chat_id} --content {content}
```

## 📋 信息提取规则

### 需要自动提取

| 信息类型 | 提取内容 |
|----------|----------|
| **讨论主题** | 讨论的核心议题 |
| **结论** | 讨论最终得出的结论 |
| **待办任务** | 需要后续跟进完成的任务 |
| **会议安排** | 确定了时间的会议/活动，自动创建视频会议 |
| **时间节点** | 确定了时间的截稿/截止日期（支持多人不同截止日期） |
| **责任人** | 谁负责哪个任务/哪个会议 |
| **参考链接** | 讨论中提到的参考资料链接 |

### 责任人匹配处理

对于昵称和飞书显示名不一致的问题：

1. **优先匹配**：AI 根据上下文语义理解，尝试匹配群成员
2. **不确定就问**：如果匹配度低于阈值，主动列出候选人请求用户确认
3. **确认后执行**：用户确认后再创建日历和@提醒

这样保证准确性，不会乱@错人。

### 多截止日期处理

如果讨论中不同责任人有不同截止日期：

- 分别提取每个任务 + 责任人 + 截止日期
- 分别为每个责任人创建独立的日历提醒
- 分别@责任人，每个人只看到自己的任务

完美支持复杂的分工场景。

### 输出格式规范

**飞书文档**结构：
```markdown
# 【每周选题】2026年X月X周

## 选题一览表
| 选题 | 责任人 | 截止时间 | 状态 | 链接 |
|------|--------|----------|------|------|
| ... | ... | ... | ... | ... |

## 详细内容
### 选题一：XXX
- **讨论结论**: ...
- **待办**: ...
- **责任人**: ...
```

**多维表格**字段：
| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | 文本 | 议题/任务标题 |
| 责任人 | 人员 | 负责人 |
| 截止时间 | 日期 | 截止/开始时间 |
| 状态 | 单选 | 待开始/进行中/已完成 |
| 文档链接 | 链接 | 完整文档链接 |

## 🚀 快速开始

### 前置要求

- 已安装 [飞书 CLI](https://github.com/larksuite/cli) 并完成认证
- OpenClaw 龙虾已接入飞书机器人
- 机器人已加入需要使用的群聊

### 配置环境变量

```bash
# 默认配置
export LARK_COLLAB_DEFAULT_DOC_FOLDER=your_folder_token
export LARK_COLLAB_DEFAULT_BITABLE_APP=your_app_token
export LARK_COLLAB_DEFAULT_BITABLE_TABLE=your_table_id
```

### 在群聊中使用

讨论完成后，在群里：

```
@lark-collab 帮我整理今天的讨论，生成选题文档
```

或者：

```
@lark-collab 整理今天的会议纪要，分配任务
```

lark-collab 自动走完整个流程，输出结果。

## 🔒 安全边界

### 设计原则
- 只处理你**触发后**指定群聊的历史消息
- 所有数据**只存在你的飞书**，不会上传到第三方
- 只新增不删除，不会修改已有内容，安全可靠

### 禁止操作
- ❌ 不会读取未授权的群聊/私聊
- ❌ 不会向外泄露群聊内容
- ❌ 不会删除任何内容

### 需要确认
- ⚠️ 创建多个日程需要确认
- ⚠️ 责任人匹配不确定需要确认
- ⚠️ 写入多维表格需要确认

## 💡 设计哲学 - 少即是多，通用基座

### 核心思想

**飞书已经有完整的协作能力**，lark-collab 只做「**流程胶水层**」：

- ✅ AI 理解讨论内容
- ✅ 调用飞书 API 自动创建
- ✅ 把分散的手动步骤串成一键完成

**不重复造轮子**：
- 📄 文档 → 用飞书文档
- 📊 表格 → 用飞书多维表格
- 📅 日历 → 用飞书日历
- 🎥 视频会议 → 用飞书视频会议
- 💬 聊天 → 用飞书群聊

### 通用基座，支持任意场景

这不是一个只能做特定事情的工具，而是一个**通用协作流程基座**：

- 你讨论什么，它就整理什么
- 你决定内容，它负责自动化执行
- 支持选题讨论、会议纪要、需求评审、活动收集、周报汇总... **任何协作场景都能用**
- 所有内容、成员、时间都由你的讨论决定，不需要改代码就能适配

**好处**：
- 保持最小核心，不臃肿
- 利用飞书原生体验，用户本来就会用
- 基座可扩展，不同团队可以在此基础上定制自己的流程
- **选择权始终留给用户**：提取结果先确认，确认后再创建，不会擅自替你做决定
- 用户完全可控，所有数据都在你的飞书里
- 个性化定制：用户可以自定义输出结构、表格字段、提醒时间
