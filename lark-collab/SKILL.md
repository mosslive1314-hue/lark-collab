# lark-collab v1.0.0 - 飞书协作流

> **飞书群聊协作全流程自动化基座** — 从讨论到产出，自动帮你完成，减少手动重复劳动。

## 🎯 定位

**lark-collab 是一个**飞书协作自动化基座：

- 你在飞书群聊讨论完，它自动帮你**整理信息 → 生成文档 → 创建会议/日历 → 分配任务**
- 利用飞书原生能力（文档/多维表格/日历/视频会议/机器人），不重复造轮子
- **减少手动重复工作**，让你专注讨论，不用事后整理

**适合场景**：
- ✅ 团队选题讨论 → 自动整理选题库 → 分配任务 → 创建截稿日历
- ✅ 项目会议约时间 → 确定时间后**一键创建飞书视频会议** → 邀请所有人 → 发送会议链接
- ✅ 每日行业信息收集 → 自动分类整理 → 生成周报文档
- ✅ 项目会议讨论 → 自动整理纪要 → 提取待办 → 创建跟进日历
- ✅ 社群活动收集 → 自动整理活动信息 → 写入多维表格 → 创建活动日历提醒

## 🧠 策略引擎

### 核心工作流

```
  群聊讨论完成
    ↓
@lark-collab 触发整理
    ↓
  1. 读取讨论时间段的群聊历史
  2. AI 提取结构化信息（结论/待办/会议/时间/责任人）
  3. 生成飞书文档（结构化格式）
  4. 数据写入飞书多维表格（方便统计筛选）
  5. 识别出确定时间的事项 → 自动创建飞书日历，**自动生成飞书视频会议链接**
  6. @ 相关责任人，发送提醒和会议链接
  7. 输出文档链接和总结到群聊
    ↓
  ✅ 完成！所有结果都在飞书里，不需要跳转
```

## ✨ 核心能力

| 能力 | 说明 |
|------|------|
| 📖 **读取群聊历史** | 获取指定时间段/条数的群聊讨论上下文 |
| 🧹 **自动结构化整理** | 从散乱聊天提取结构化信息：结论/待办/会议/时间/责任人 |
| 📄 **生成飞书文档** | 整理结果生成格式化飞书文档，可分享 |
| 📊 **写入多维表格** | 结构化数据存入多维表格，方便后续筛选统计 |
| 📅 **自动创建日历 + 视频会议** | 从内容识别时间和事项，自动创建飞书日历，**自动生成飞书视频会议链接** |
| 👥 **任务自动分配** | 识别责任人，@ 相关成员发送提醒 |
| 🔔 **自动提醒** | 会议开始前提醒，日历提前提醒，截止日期提醒 |

## 🛠️ 工具调用（通过飞书 CLI）

```bash
# 1. 读取群聊历史
lark-cli im messages list --chat-id {chat_id} --limit {limit}

# 2. 创建飞书文档
lark-cli doc create --title {title} --content {markdown_content}

# 3. 写入多维表格
lark-cli bitable record add --app {app_token} --table {table_id} --fields {json}

# 4. 创建日历 + 自动生成视频会议
# 默认自动创建飞书视频会议，不需要额外配置
lark-cli calendar event create --summary {summary} --start {start} --end {end} --attendees {emails} --vchat

# 5. 发送群消息
lark-cli im message send --chat-id {chat_id} --content {content}
```

## 📋 信息提取规则

### 需要自动提取

| 信息类型 | 提取内容 |
|----------|----------|
| **讨论主题** | 讨论的核心议题 |
| **结论** | 讨论最终得出的结论 |
| **待办任务** | 需要后续跟进完成的任务 |
| **会议安排** | 确定了时间的会议/活动，自动创建视频会议 |
| **时间节点** | 确定了时间的截稿/截止日期 |
| **责任人** | 谁负责哪个任务/哪个会议 |
| **参考链接** | 讨论中提到的参考资料链接 |

### 输出格式规范

**飞书文档**结构：
```markdown
# 【每周选题】2026年X月X周

## 选题一览表
| 选题 | 责任人 | 截止时间 | 状态 | 链接 |
|------|--------|----------|------|------|
| ... | ... | ... | ... | ... |

## 详细内容
### 选题一：XXX
- **讨论结论**: ...
- **待办**: ...
- **责任人**: ...
```

**多维表格**字段：
| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | 文本 | 议题/任务标题 |
| 责任人 | 人员 | 负责人 |
| 截止时间 | 日期 | 截止/开始时间 |
| 状态 | 单选 | 待开始/进行中/已完成 |
| 文档链接 | 链接 | 完整文档链接 |

## 🚀 快速开始

### 前置要求

- 已安装 [飞书 CLI](https://github.com/larksuite/cli) 并完成认证
- OpenClaw 龙虾已接入飞书机器人
- 机器人已加入需要使用的群聊

### 配置环境变量

```bash
# 默认配置
export LARK_FLOW_DEFAULT_DOC_FOLDER=your_folder_token
export LARK_FLOW_DEFAULT_BITABLE_APP=your_app_token
export LARK_FLOW_DEFAULT_BITABLE_TABLE=your_table_id
```

### 在群聊中使用

讨论完成后，在群里：

```
@lark-flow 帮我整理今天的讨论，生成选题文档
```

或者：

```
@lark-flow 整理今天的会议纪要，分配任务
```

机器人自动走完整个流程，输出结果。

## 🔒 安全边界

### 设计原则
- 只处理你**触发后**指定群聊的历史消息
- 所有数据**只存在你的飞书**，不上传到第三方
- 不会修改/删除已有内容，只新增

### 禁止操作
- ❌ 不会读取未触发的群聊/私聊
- ❌ 不会删除任何已有内容
- ❌ 不会向外分享群聊内容

### 需要确认
- ⚠️ 创建多个日程需要确认
- ⚠️ 写入多维表格需要确认

## 💡 设计哲学 - 少即是多，基座设计

### 核心思想

**飞书已经做好了文档/表格/日历/IM，lark-flow 只做**：

1. **AI 理解** - 从聊天读懂你想要什么
2. **API 调用** - 调用飞书 CLI 自动创建
3. **流程编排** - 把分散步骤串成一键完成

**不重复造轮子**：
- ✗ 不会重新实现文档编辑器
- ✗ 不会重新实现日历App
- ✗ 不会重新实现IM聊天

**保持**：
- **最小完备** - 只做胶水层，连接 AI 和飞书原生能力
- **渐进披露** - 复杂配置可选，默认够用
- **基座可扩展** - 不同团队可以在这个基座上定制自己的流程
