## ✨ 核心能力

| 能力 | 说明 |
|------|------|
| 📖 **读取群聊历史** | 获取讨论上下文，AI 深度理解讨论内容 |
| 📝 **会议纪要自动整理** | 会议结束后，自动整理讨论结论、待办任务、决策事项 |
| 🧹 **自动结构化整理** | 从散乱聊天提取结构化信息：结论/待办/会议/时间/责任人 |
| 📄 **生成飞书文档** | 整理结果自动生成格式化飞书文档，可分享 |

| 📅 **自动创建日历 + 视频会议** | 从内容识别时间和事项，自动创建飞书日历，**自动生成飞书视频会议链接** |
| 👥 **任务自动分配** | 识别责任人，每个任务不同截止日期分别提醒，昵称不确定时主动请求确认 |
| 🔔 **自动提醒** | 会议开始前提醒，日历提前提醒，每个任务截止日期分别提醒 |

## 🗺️ 完整工作流

```
  群聊讨论完成（会议/选题/项目讨论）
    ↓
@lark-collab 触发整理
    ↓
  1. 读取讨论时间段的完整群聊历史
  2. AI 深度理解，提取结构化信息：
     - 讨论主题和结论
     - 需要跟进的待办任务
     - 需要召开的会议（时间/参会人）
     - 每个任务的责任人 + 截止日期
  3. 生成完整的飞书文档（会议纪要/选题列表）
  4. 结构化数据写入飞书多维表格（方便筛选跟踪）
  5. 自动创建日历事件：
     - 会议 → 自动生成飞书视频会议链接
     - 截止日期 → 每个责任人不同时间分别创建提醒
  6. @ 相关责任人，发送提醒和链接
     - 昵称匹配不确定时 → 主动请求用户确认
  7. 输出完整文档链接到群聊
    ↓
  ✅ 完成！从讨论到落地，全闭环自动化
```

## 💡 关键问题处理

### 🎯 讨论了初步结果，不一定想立马创建怎么办？

**lark-collab 的设计**：
- 先提取信息，生成文档
- **列出提取结果，请求你确认**
- 你确认要创建哪些，再执行创建
- **选择权始终留给你**，不会擅自替你做决定

避免误创建，避免你手动取消的麻烦。

### ⏰ 可以自定义提醒时间吗？

完全可以！✨

- 创建会议/任务时，你可以说 "提前三天提醒我"
- 支持：提前 N 天/提前 N 小时提醒
- 不同会议/任务可以有不同的提醒时间
- 默认使用配置，你自定义就按你说的来

### 昵称和飞书显示名不一致怎么办？

lark-collab 会：
1. AI 根据上下文语义尝试匹配群成员
2. **匹配度不够就问你** → 主动列出候选人让你确认
3. 确认后再执行，不会乱@错人

### 不同责任人有不同截止日期怎么办？

- 分别提取每个任务 + 责任人 + 截止日期
- 分别为每个人创建独立的日历提醒
- 分别@责任人，每个人只看到自己的任务

完美支持复杂分工。

### 🔄 文档更新后，可以同步新增任务吗？

完全支持！💪

团队共享文档讨论更新 → 讨论确定新增任务 → @lark-collab 同步
- 从讨论中提取新增任务
- 更新项目多维表格
- 给责任人创建提醒
- 发送更新列表到群里

## 💡 设计哲学 - 少即是多，通用基座

### 核心思想

**飞书已经有完整的协作能力**，lark-collab 只做「**流程胶水层**」：

- ✅ AI 理解讨论内容
- ✅ 调用飞书 API 自动创建
- ✅ 把分散的手动步骤串成一键完成

**不重复造轮子**：
- 📄 文档 → 用飞书文档
- 📊 表格 → 用飞书多维表格
- 📅 日历 → 用飞书日历
- 🎥 视频会议 → 用飞书视频会议
- 💬 聊天 → 用飞书群聊

### 通用基座，支持任意场景

这不是一个只能做特定事情的工具，而是一个**通用协作流程基座**：

- 你讨论什么，它就整理什么
- 你决定内容，它负责自动化执行
- 支持选题讨论、会议纪要、需求评审、活动收集、周报汇总... **任何协作场景都能用**
- 所有内容、成员、时间都由你的讨论决定，**不需要改代码就能适配**

## 🚀 快速开始

### 前置要求

- 已安装 [飞书 CLI](https://github.com/larksuite/cli) 并完成认证
- OpenClaw 龙虾已接入飞书机器人
- 机器人已加入目标群聊

### 配置环境变量

```bash
# 默认配置
export LARK_COLLAB_DEFAULT_DOC_FOLDER=your_folder_token
export LARK_COLLAB_DEFAULT_BITABLE_APP=your_app_token
export LARK_COLLAB_DEFAULT_BITABLE_TABLE=your_table_id
```

### 在群聊中使用

讨论完成后，直接在群里 @ 机器人：

```
@lark-collab 帮我整理今天的讨论，生成选题文档
```

或者：

```
@lark-collab 整理今天的会议纪要，分配任务
```

lark-collab 自动走完整个流程，输出结果。

## 🔒 安全边界

- 只处理触发时指定群聊的历史消息
- 所有数据存储在你的飞书，不会向外传播
- 只新增不删除，安全可靠
- 不确定的事情会主动问你，不会瞎猜
