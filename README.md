## 🎯 解决什么问题

团队在飞书群聊讨论完，你还在手动做这些吗？

- ❌ 手动复制粘贴整理讨论结论和会议纪要
- ❌ 手动创建表格统计选题和任务
- ❌ 手动创建日历邀请，生成视频会议链接
- ❌ 手动 @ 每个人提醒任务和截止日期
- ❌ 不同责任人不同截止日期，要一个个创建

**这些重复劳动，lark-collab 一键帮你搞定** ✅

讨论完一句话触发，全流程自动化完成。

## ✨ 核心能力

| 能力 | 说明 |
|------|------|
| 📖 **读取群聊历史** | 获取讨论上下文，AI 深度理解讨论内容 |
| 📝 **会议纪要自动整理** | 会议结束后，自动整理讨论结论、待办任务、决策事项 |
| 🧹 **自动结构化整理** | 从散乱聊天提取结构化信息：结论/待办/会议/时间/责任人 |
| 📄 **生成飞书文档** | 整理结果自动生成格式化飞书文档，可分享 |
| 📊 **写入多维表格** | 结构化数据存入多维表格，方便后续筛选跟踪 |
| 📅 **自动创建日历 + 视频会议** | 从内容识别时间和事项，自动创建飞书日历，**自动生成飞书视频会议链接** |
| 👥 **任务自动分配** | 识别责任人，每个任务不同截止日期分别提醒，昵称不确定时主动请求确认 |
| 🔔 **自动提醒** | 会议开始前提醒，日历提前提醒，每个任务截止日期分别提醒 |

## 🗺️ 完整工作流

```
  群聊讨论完成（会议/选题/项目讨论）
    ↓
@lark-collab 触发整理
    ↓
  1. 读取讨论时间段的完整群聊历史
  2. AI 深度理解，提取结构化信息：
     - 讨论主题和结论
     - 需要跟进的待办任务
     - 需要召开的会议（时间/参会人）
     - 每个任务的责任人 + 截止日期
  3. 生成完整的飞书文档（会议纪要/选题列表）
  4. 结构化数据写入飞书多维表格（方便筛选跟踪）
  5. 自动创建日历事件：
     - 会议 → 自动生成飞书视频会议链接
     - 截止日期 → 每个责任人不同时间分别创建提醒
  6. @ 相关责任人，发送提醒和链接
     - 昵称匹配不确定时 → 主动请求用户确认
  7. 输出完整文档链接到群聊
    ↓
  ✅ 完成！从讨论到落地，全闭环自动化
```

## 🎯 常见使用场景

### 1. 📝 编辑团队每周选题会

**讨论完选题** → @lark-collab 整理

自动完成：
1. 整理所有确认的选题
2. 生成选题表飞书文档
3. 写入选题库多维表格
4. 给认领选题的编辑创建截稿日历
5. 不同编辑不同截止日期，分别提醒
6. @ 编辑提醒

**你**不用：
- ✗ 手动复制粘贴
- ✗ 手动做表格
- ✗ 手动创建日历
- ✗ 挨个提醒

**lark-collab** 一键搞定 🎉

2. 创建飞书日历邀请
3. **自动生成飞书视频会议链接**

5. 发送会议链接到群里

自动完成：
1. 收集所有信息
2. 按分类整理
3. 生成日报飞书文档
4. 发送文档链接到群

### 5. 🎪 社群活动收集

1. 整理所有活动信息（时间/地点/报名链接）

3. 写入多维表格
4. 给每个活动创建日历提醒

## 💡 关键问题处理

### 昵称和飞书显示名不一致怎么办？

lark-collab 会：
1. AI 根据上下文语义尝试匹配群成员
2. **匹配度不够就问你** → 主动列出候选人让你确认
3. 确认后再执行，不会乱@错人

### 不同责任人有不同截止日期怎么办？

- 分别提取每个任务 + 责任人 + 截止日期
- 分别为每个人创建独立的日历提醒
- 分别@责任人，每个人只看到自己的任务

完美支持复杂分工。

## 💡 设计哲学 - 少即是多，通用基座

### 核心思想

**飞书已经有完整的协作能力**，lark-collab 只做「**流程胶水层**」：

- ✅ AI 理解讨论内容
- ✅ 调用飞书 API 自动创建
- ✅ 把分散的手动步骤串成一键完成

**不重复造轮子**：
- 📄 文档 → 用飞书文档
- 📊 表格 → 用飞书多维表格
- 📅 日历 → 用飞书日历
- 🎥 视频会议 → 用飞书视频会议
- 💬 聊天 → 用飞书群聊

### 通用基座，支持任意场景

这不是一个只能做特定事情的工具，而是一个**通用协作流程基座**：

- 你讨论什么，它就整理什么
- 你决定内容，它负责自动化执行
- 支持选题讨论、会议纪要、需求评审、活动收集、周报汇总... **任何协作场景都能用**
- 所有内容、成员、时间都由你的讨论决定，**不需要改代码就能适配**

## 🚀 快速开始

### 前置要求

- 已安装 [飞书 CLI](https://github.com/larksuite/cli) 并完成认证
- OpenClaw 龙虾已接入飞书机器人
- 机器人已加入目标群聊

### 配置环境变量

```bash
# 默认配置
export LARK_COLLAB_DEFAULT_DOC_FOLDER=your_folder_token
export LARK_COLLAB_DEFAULT_BITABLE_APP=your_app_token
export LARK_COLLAB_DEFAULT_BITABLE_TABLE=your_table_id
```

### 在群聊中使用

讨论完成后，直接在群里 @ 机器人：

```
@lark-collab 帮我整理今天的讨论，生成选题文档
```

或者：

```
@lark-collab 整理今天的会议纪要，分配任务
```

lark-collab 自动走完整个流程，输出结果。

## 🔒 安全边界

- 只处理触发时指定群聊的历史消息
- 所有数据存储在你的飞书，不会向外传播
- 只新增不删除，安全可靠
- 不确定的事情会主动问你，不会瞎猜
