## 🎯 解决什么问题

团队在飞书群聊讨论完，你还在手动做这些吗？

- ❌ 手动复制粘贴整理讨论结论
- ❌ 手动创建表格统计选题
- ❌ 手动创建日历邀请
- ❌ 手动 @ 每个人提醒

**这些重复劳动，lark-collab 一键帮你搞定** ✅

---

## ✨ 核心能力

| 能力 | 说明 |
|------|------|
| 📖 **读取群聊历史** | 获取讨论上下文，AI 理解讨论内容 |
| 🧹 **自动结构化整理** | 从散乱聊天提取结构化信息：结论/待办/时间/责任人 |
| 📄 **生成飞书文档** | 整理结果自动生成格式化飞书文档 |
| 📊 **写入多维表格** | 结构化数据存入多维表格，方便后续筛选统计 |
| 📅 **自动创建日历** | 从内容识别时间和事项，自动创建飞书日历邀请 |
| 👥 **任务自动分配** | 识别责任人，@ 相关成员发送提醒 |
| 🔔 **自动提醒** | 日程提前提醒，截止日期提醒 |

---

## 🗺️ 工作流

```
  群聊讨论完成
    ↓
@lark-collab 触发整理
    ↓
  1. 读取讨论时间段的群聊历史
  2. AI 提取结构化信息（结论/待办/时间/责任人）
  3. 生成飞书文档（结构化格式）
  4. 数据写入飞书多维表格
  . 识别确定时间的事项 → 自动创建飞书日历
  6. @ 责任人发送提醒
  7. 输出文档链接和总结到群聊
    ↓
  ✅ 完成！结果全都在飞书里，不需要跳转
```

## 🚀 快速开始

### 前置要求

- 已安装 [飞书 CLI](https://github.com/larksuite/cli) 并完成认证
- OpenClaw 龙虾已接入飞书机器人
- 机器人已加入目标群聊

### 配置环境变量

```bash
# 默认配置
export LARK_COLLAB_DEFAULT_DOC_FOLDER=your_folder_token
export LARK_COLLAB_DEFAULT_BITABLE_APP=your_app_token
export LARK_COLLAB_DEFAULT_BITABLE_TABLE=your_table_id
```

### 在群聊中使用

讨论完成后，直接在群里 @ 机器人：

```
@lark-collab 帮我整理今天的讨论，生成选题文档
```

或者：

```
@lark-collab 整理今天的会议纪要，分配任务
```

lark-collab 自动走完全流程，输出结果。

## 💡 设计哲学 - 少即是多，基座设计

### 核心思想

**飞书已经有完整的协作能力**，lark-collab 只做「胶水层」：

- ✅ AI 理解讨论内容
- ✅ 调用飞书 API 自动创建
- ✅ 把分散步骤串成一键完成

**不重复造轮子**：
- 📄 文档 → 用飞书文档
- 📊 表格 → 用飞书多维表格
- 📅 日历 → 用飞书日历
- 💬 聊天 → 用飞书群聊

**好处**：
- 保持最小核心，不臃肿
- 利用飞书原生体验，用户本来就会用
- 基座可扩展，不同团队可以在此基础上定制自己的流程

---

## 🔒 安全边界

- 只处理触发时指定群聊的历史消息
- 所有数据存储在你的飞书，不会向外传播
- 只新增不删除，安全可靠
